# 指値・逆指値計算アプリ

株式トレードのエントリー価格と許容リスクから、利確価格・損切り価格・推奨購入株数をシミュレーションするWebアプリです。複数の計算方式を並べて比較でき、資金管理を考慮した売買プラン作成を支援します。

Google AI Studioを開発支援に活用しながら、要件定義、計算ロジックの設計、UI構成、コードの調整、動作確認を行って制作しました。AIが生成したものをそのまま使うのではなく、トレード時に必要な情報を整理し、比較しやすさと使いやすさを意識して仕上げています。

> [!IMPORTANT]
> 本アプリの計算結果は参考情報です。特定の銘柄や売買を推奨するものではなく、利益を保証するものでもありません。実際の投資判断はご自身の責任で行ってください。

## 制作背景

株式トレードでは、エントリー前に「どこで損切りするか」「どこで利益を確定するか」「何株まで購入できるか」を決める必要があります。しかし、それぞれを手計算しながら複数の戦略を比較するのは手間がかかります。

そこで、必要な数値を入力するだけで売買プランを可視化し、感覚ではなくリスクを基準に判断できるツールを目指して本アプリを制作しました。

## 開発で意識したポイント

- **リスク管理を中心に設計**：利益額だけでなく、1回の取引で許容できる損失額から購入株数を逆算します。
- **複数の判断基準を比較可能に**：リスクリワード比、サポート／レジスタンス、ATR、直近高安を同じ画面で比較できます。
- **入力と結果を一画面に集約**：数値を変更すると結果へすぐ反映され、条件を変えながら検討できます。
- **実際の利用場面を想定**：日本株を想定し、推奨株数は100株単位かつ許容損失額を超えないよう切り捨てています。
- **保守しやすい構成**：UIコンポーネント、型定義、計算ロジックを分離し、機能を追加しやすくしています。

## 主な機能

- 買い・売りの両方向に対応
- 利確価格と損切り価格をリアルタイム計算
- 4種類の計算方式を比較
  - 許容損失率・リスクリワード比
  - サポート・レジスタンス
  - ATR（Average True Range）
  - 直近高値・直近安値
- 運用資金と許容損失額から推奨株数を算出
- 想定最大利益・想定最大損失を表示
- デイトレード／スイングトレード用の入力プリセット
- レスポンシブ対応

## 計算の概要

### 許容損失率・リスクリワード比

買いの場合は、次の考え方で価格を算出します。

```text
損切り価格 = エントリー価格 × (1 - 許容損失率 / 100)
利確価格   = エントリー価格 + 1株あたりの損失額 × リスクリワード比
```

売りの場合は、価格の上下を反転して計算します。

### ATR

エントリー価格から `ATR × 2` 離れた位置を損切り価格とし、指定したリスクリワード比から利確価格を算出します。

### 推奨株数

```text
許容損失額 = 運用資金 × 許容損失率 / 100
推奨株数   = 許容損失額 ÷ 1株あたりの損失額
```

推奨株数は、許容損失額を超えないよう100株単位で切り捨てます。許容損失額を直接入力した場合は、その値が優先されます。

## セットアップ

### 必要な環境

- Node.js 18以上
- npm

### インストールと起動

```bash
git clone <このリポジトリのURL>
cd sashine-keisan
npm install
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開いてください。

現在の計算機能はブラウザ内で完結するため、APIキーなどの環境変数は不要です。

## 使い方

1. 売買方向（買い／売り）を選択します。
2. エントリー価格、リスクリワード比、許容損失率を入力します。
3. 必要に応じてサポート、レジスタンス、ATR、直近高値・安値を入力します。
4. 運用資金または1回あたりの許容損失額を入力します。
5. 表示された戦略を選び、利確価格・損切り価格・推奨株数を比較します。
